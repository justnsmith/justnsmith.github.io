## I. Introduction: 

Artificial Intelligence has become an increasingly useful tool in education, particularly in Software Engineering. They can automate tasks and provide real-time assistance while also supporting personalized learning which has made it a valuable asset in the classroom. In ICS 314, the use of AI technologies has been a key component in my learning experience. Through tools like ChatGPT and GitHub Copilot, I have been able to get instant feedback, generate code examples, and get explanations on code that I did not understand. These AI tools have not only changed the way I learn software engineering principles but it has also changed how I approach solving problems and coding tasks in the course. In this paper, I will reflect on my use and experience of AI in ICS 314 by looking at specific instances where AI has been most useful, the challenges I encountered, and the potential for AI to change the future of software engineering education.

## II. Personal Experience with AI:
<br/>

### 1. Experience WODs:

I used AI for a few of the experience WODs, mostly with the functional programming ones. This is because I understood what a function had to do and could write it in a nonfunctional way, so I used AI to turn this nonfunctional code into a functional way. Most of the experience WODs had a screencast that went over the solution which I used mostly.

### 2. In-class Practice WODs:

I also only used AI for the functional programming WODs due to my not knowing how to program functionally due to all the methods that I would need to memorize. I would write the code nonfunctional and then use ChatGPT to turn it into a functional way.

### 3. In-class WODs:

I used ChatGPT and GitHub Copilot for a majority of these WODs. The first couple of WODs were fairly easy which required no AI usage. However, once we moved on to functional programming and all the concepts after that. I relied on AI to assist me with checking my code to make sure it was written and helping me fix syntax issues and debugging. I remember once we moved on to actual web development, I relied heavily on AI to help with styling and generating code. I would often ask the AI to provide code that I used as a template for these web development WODs, then I would use this code it produced as a template and work around that to get the desired outcome.

### 4. Essays: 

I did not use AI that much for the essays. On some of the essays, I would ask it to come up with ideas to talk about, but other than that the writing was done by me. For grammar checking, I used Grammarly to provide input on how I could improve my writing. 

### 5. Final Project: 

I used ChatGPT heavily during the final project. I felt like we moved so quickly through concepts that it was hard for me to get a solid understanding of everything to succeed in the final project. I used ChatGPT to make templates for the mock-up pages and modified them myself to get the desired output. In the final project, I was mostly working on the sign-in, sign-out, sign-up, forgot password, etc. pages. I would use AI to originally make a template of these pages and would modify them to how I like. Then sometimes I found that it looked quite simple so I would ask ChatGPT to make it more ‘modern’ and ‘look nicer’ and it would come up with several options and I picked the one that I found looked the best. Then I asked it to match the theme of the other pages and it did so. I also used ChatGPT heavily to create the email system within the final project. It didn’t work smoothly, it took me several hours and I had to watch a video on YouTube of how to get it to work simply. 

### 6. Learning a Concept/Tutorial:

I don’t remember specifically using AI to teach me a concept, however, I definitely have learned through ChatGPT just by asking it questions on how to do certain things for the class. For example, in my final project, I had no idea how to set up email verification. After prompting ChatGPT to give me ideas on how I could do this, I learned about nodemailer and did research on it and learned about it. 

### 7. Answering a Question in Class or Discord:

Did not use AI to answer a question in class or in Discord. This is because, in class, it felt like even 
though it was encouraged to answer questions using AI, the teacher would still ask follow-up questions to the student. So I avoided answering questions in general. I also didn’t answer questions in Discord since I mainly just used it as a source of information and communication with my teammates. I was never active on the questions channel.

### 8. Asking or Answering a Smart-Question:

Same answer as above. I was not active in the smart-question channel so I didn’t use AI to answer questions as I didn’t answer any questions at all.

### 9. Coding Example:

I did not use coding examples that much for ICS 314. I somewhat used it to make templates for my final project mock-up pages but I wouldn’t classify it as an example. The prompt I would use would ask the AI to make a mockup page containing certain components like buttons or containers and where roughly to position them. Then I would change up the positioning, sizes, and style to my liking.

### 10. Explaining Code:

I rarely used AI to explain code. I think I used it once to explain a functional programming concept that I saw in a piece of code. My prompt went along the lines of “Please explain what the following piece of code does:”. It was useful in explaining what it did and I got a better understanding of how functional programming works.

### 11. Writing Code:

Yes I used AI heavily to write my code during most of the ending in-class WODs and my final project. I would prompt the AI with my specifications and requirements and would test what it would produce. If I needed changes I would keep prompting the AI until I got my desired output.

### 12. Documenting Code:

I did not directly use AI to document code. However, the AI documented the code it wrote by itself in comments to explain to me what it does. I used these explanations for my teammates to also understand what the code does so in a way I did use AI to document code. I would not prompt the AI with anything special to get documentation. I found this useful rather than having to write my documentation and explain what the code does.

### 13. Quality Assurance:

Yes, I used ChatGPT a ton to fix errors and debug. Oftentimes when working on the final project, I would get errors on the web page and would just copy and paste the error into ChatGPT with some context and the code that was involved in the issue. Most of the time it was able to detect what was wrong and fix it. There were a few times when I wasn’t able to come up with the fix, I would have prompted the AI several times and ended up having to figure it out on my own. For example, the email verification system took me a very long time. I tried to use AI to initially set it up but many errors resulted from this. I had to watch a YouTube tutorial and follow that to initially get it working, and after that, ChatGPT helped me finish developing the system.

### 14. Other Uses in ICS 314 Not Listed:

I did use AI to help me with writing a message to the TA asking for them to look at my WOD submission again since I felt like my grade was incorrect. I initially wrote up a message and then asked AI to make it sound nicer and give some context. After it produced a message, I read it and changed my initial writing to contain some of these nicer elements to make my message more appealing.

## III. Impact on Learning and Understanding:

I think the overall incorporation of AI has influenced my learning experience in a negative way. Yes, AI does increase my work productivity by helping me be able to complete tasks quickly and efficiently. However, I think I rely too heavily on AI to the point where it takes away from my learning. I believe if AI wasn't a thing and that I had to figure out everything on my own throughout this class, I would’ve taken more out of it. I think I would’ve struggled more and maybe my grades wouldn’t have been as good, but I would’ve learned more. I think this is also true for other classes. A lot of the time I won’t bother doing some tasks since I know I can just ask AI to do it for me. 

To me, AI has a negative impact on my skill development and problem-solving abilities. I feel like if I didn’t use AI at all, I would’ve developed more skills and had more time to develop my problem-solving abilities since I would’ve struggled more and spent more time on certain problems. AI has also challenged my understanding of software engineering concepts since I didn’t need to fully understand the concepts, I would just ask the AI if I didn’t understand something and it would go ahead and explain it to me. I wouldn’t memorize these concepts and it wouldn’t stick with me.

## IV. Practical Applications:

I believe that there are practical applications for AI outside of ICS 314. I’ve used the same AI learning approach to a personal project that I have been working on. It’s about creating a blockchain from scratch. I used ChatGPT to explain to me how a blockchain works and the inner workings of it. I used AI to tell me what I needed to create in order to get a functional blockchain and would use example code to help start me off. I think that AI can be very effective in real-world software engineering challenges, but I do think that AI starts to struggle once the issue becomes something large and not easy to explain.

## V. Challenges and Opportunities:

One challenge and limitation that I faced when using AI during ICS 314 was during the in-class WOD where we would have to make a web page look like an image that was provided. Since the AI can only take text input, it was hard to describe to the AI what I wanted the web page to look like. This caused me issues since it would often come up with code but not what I needed it to be, which would cause me to prompt it and tell it to change slightly. When it couldn’t get the result I wanted, I had to ask it how I could change it myself so I could play around with it to meet the requirements of the example image. 

Another challenge I faced was during the final project with the email verification system. The AI had a hard time figuring out how to set this up in the next js project since it didn’t know the folder structure. I ended up having to follow a tutorial to get it initially set up and then ChatGPT was able to help me further with getting everything solidified.

I do think that AI has potential opportunities to help students learn about software engineering. This needs to be done carefully though because the use of too much AI takes away from the education due to the students becoming too reliant on the AI. This is what I found happened to me and if I could take this class again, I would be less reliant on AI and try harder to understand these concepts by myself.

## VI. Comparative Analysis:

Traditional learning methods in software engineering consist of learning through trial and error, which helps with engagement and remembering concepts. This helps improve problem-solving and increases your coding skills. This approach encourages students to dig into the material. On the other hand, AI tools make learning much faster by providing immediate help, whether it's generating code or explaining unknown concepts. But, if you rely too much on AI, it can feel as if you’re skipping through the tough parts, which may hurt long-term understanding and problem-solving skills. AI does help with practical skills, but traditional methods may be better for keeping you engaged and learning the content.

## VII. Future Considerations:

I think that AI will play a big role in the future of software engineering education. As these tools get better, they could provide more personalized learning which offers tailored feedback and helps with more complex tasks. But, if the students lean too much on AI, they will miss out on actually learning and understanding the material. Ideally, future AI tools should push the students to think more about why something works, and not just give them the answer right away. This could help keep the balance of learning and understanding while also getting tailored help.

## VIII. Conclusion:

AI tools like ChatGPT and GitHub Copilot have made coding and problem-solving easier for me, especially when I am stuck. But, I think I relied on them too much, which stopped me from truly understanding some concepts. AI can be useful in a class like this, but it shouldn’t replace thinking through problems. In future courses, it would be great to use AI as a backup and not as a crutch so that we can get the most out of the learning experience.
